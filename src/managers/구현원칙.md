## 아키텍처 전략
우리 프로젝트는 **"선언적 상태 관리(Zustand)와 명령형 3D 로직(Three.js/Manager)의 결합"**을 핵심 전략으로 취합니다.

- **Single Source of Truth (SSOT)**: 모든 데이터의 근간은 Zustand 스토어(`useObjectStore`)입니다.
- **Bridge Pattern**: 스토어의 상태 변경을 감지하여 3D 씬을 업데이트하거나, 반대로 3D 씬에서의 인터랙션을 상태에 반영하는 중간 매개체로 '매니저'들을 활용합니다.
- **관심사 분리 (SoC)**: UI 로직, 3D 렌더링 세부 사항, 비즈니스 로직을 분리하여 각 모듈의 독립성을 보장합니다.

## 구현 원칙
- **매니저 직접 참조 지양**: 가급적 커스텀 훅(Custom Hooks)을 통해 매니저 기능에 접근합니다.
- **데이터 불변성**: 모든 데이터 수정은 반드시 Zustand 스토어의 `set()`을 통해서만 이루어져야 합니다.
- **타입 엄격 준수**: 3D 좌표 데이터 등은 `interface`를 통해 엄격하게 검증합니다.
